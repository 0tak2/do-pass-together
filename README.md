# do-pass-together

## 규칙

- 하루 두 문제 풀기
- 마감 시간은 정하지 않지만 양심껏

## 방법

- 자신의 브랜치를 만든다. `Nickname/yyyy-MM-dd` 형식
  ```
  git switch -c Bob/2025-12-27
  ```
- 풀고 싶은 문제를 푼다.
- 자신의 닉네임으로 된 디렉토리 아래에 소스코드 파일을 만들고 문제 답안을 복사하고 커밋한다. 파일 상단에 [example.py](./bob/example.py)와 같이 간단한 주석을 남긴다.
  - 소스코드 이름 형식
    - 프로그래머스: `prog-적당한-이름.py`
    - 백준: `boj-적장한-이름.py`
    - 리트코드: `leet-적장한-이름.py`
  - 커밋 메시지 형식
    - `[Isa] 2025-12-27 Assignment`
- main에 합치는 PR을 올린다.
  - PR의 제목과 형식은 템플릿을 따른다.
  - PR에 어떤 방식으로 접근했는지, 어디서 막혔는지, 어떤 자료를 참고했는지 등 자신이 나중에 돌아봤을 때 도움이 될만한 정보와 리뷰어에게 도움이 될 정보를 같이 적는다.
- 다른 사람의 PR을 보고 리뷰해준다.
- 고칠게 없으면 자기 PR을 머지한다.
